🎓 جزوه آموزشی جنگو (Django) — روز پنجم

موضوع: مرور نحوه‌ی استفاده از تمپلیت‌های HTML و مدیریت فایل‌های استاتیک در پروژه جنگو

۱) جایگزینی پاسخ HTTP با تمپلیت‌های HTML

هدف: به جای برگرداندن پاسخ متنی ساده، از فایل HTML برای نمایش صفحه به کاربر استفاده می‌شود.

from django.shortcuts import render

def home(request):
    return render(request, 'index.html')

مرسوم است که تمپلیت صفحه اصلی را index.html بنامیم.

۲) تنظیم مسیر تمپلیت‌ها

در صورت تعریف نکردن مسیر تمپلیت، با خطای TemplateDoesNotExist روبه‌رو می‌شوید.

تنظیم در فایل settings.py

'DIRS': [BASE_DIR / 'templates'],

سپس در ریشه پروژه پوشه‌ای با نام templates ایجاد کرده و فایل‌های HTML را در آن قرار دهید:

project_root/
├── manage.py
├── templates/
│   ├── index.html
│   ├── about.html
│   └── contact.html
└── app/

۳) مدیریت فایل‌های استاتیک (Static Files)

فایل‌های استاتیک شامل CSS، JS و تصاویر هستند که ظاهر صفحات را تشکیل می‌دهند.

الف) تنظیم در settings.py

STATIC_URL = 'static/'
STATICFILES_DIRS = [
    BASE_DIR / 'static',
]

ب) ساخت پوشه استاتیک

project_root/
├── static/
│   ├── css/style.css
│   ├── js/script.js
│   └── images/logo.png

ج) لینک کردن فایل‌های استاتیک در HTML

در ابتدای فایل HTML بنویسید:

{% load static %}
نوع فایلقبل از تغییربعد از تغییر
CSS<link href="css/style.css"><link href="{% static 'css/style.css' %}">
JS<script src="js/script.js"></script><script src="{% static 'js/script.js' %}"></script>
Image<img src="images/logo.png"><img src="{% static 'images/logo.png' %}" alt="logo">
نکته: لینک‌دهی فایل‌های استاتیک باید در تمام صفحات HTML (مانند about.html و contact.html) انجام شود.
